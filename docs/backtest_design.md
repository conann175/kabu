# バックテスト設計書

---

## 1. 目的

モデルが学習していない未来データに対して、期待値>1.0の馬に賭けた場合の回収率を検証します。
バックテストで回収率が安定して100%を超えることを確認してから実運用に移行します。

---

## 2. データ分割

| データセット | 期間 | 用途 |
|--------------|------|------|
| 学習データ | 2024年以前 | モデル学習 |
| 検証データ | 2025年前半 | early_stopping |
| テストデータ | 2025年後半〜 | バックテスト |

---

## 3. 回収率計算式

```
回収率(%) = 払い戻し合計 ÷ 購入金額合計 × 100
```

購入金額は100円固定で計算。

---

## 4. 評価指標

| 指標 | 合格基準 | 説明 |
|------|----------|------|
| 回収率 | 100%以上 | メインの評価指標 |
| AUC | 0.65以上 | モデルの予測精度 |
| 購入対象割合 | 10〜30% | 期待値>1.0の馬の割合 |
| 月別回収率の安定性 | 80%以上を維持 | 月ごとのばらつきが少ないか |

---

## 5. 期待値閾値別シミュレーション

```python
for threshold in [0.8, 1.0, 1.1, 1.2, 1.3, 1.5]:
    buy = df_test[df_test['expected_value'] > threshold]
    recovery = (買いの払い戻し合計) / (len(buy) * 100) * 100
    print(f'EV > {threshold}: {len(buy)}件 回収率{recovery:.1f}%')
```

---

## 6. 注意事項

- バックテストの回収率が高くても実運用では下がる場合があります
- 複勝オッズは直前に変動するため取得タイミングに注意
- 最低6ヶ月以上のテストデータで評価すること
- バックテスト合格後に実際の馬券購入に移行すること
